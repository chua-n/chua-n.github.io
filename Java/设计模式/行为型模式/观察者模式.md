观察者模式的含义为，当你在做事时旁边总有一个人盯着你，当你做的事情是他感兴趣的事情时，他就会跟着做另外一些事情，但是盯着你的人必须要到你那里登记，不然你无法通知他。

观察者模式通常包含如下几个角色：

- Subject（抽象主题）——负责管理所有观察者的引用，同时定义主要的事件操作
- ConcreteSubject（具体主题）——实现抽象主题定义的所有接口，当自己发生变化时，会通知所有观察者
- Observer（观察者）——监听主题发生变化的操作接口
